Filtering Logic

apply_filters()

Filters items by warehouse, category, and status. Skips filter if value is 'all'. Case-insensitive category matching.

filter_by_month()

Supports direct month match (2025-01) and quarter-based filtering (Q1-2025 maps to Jan/Feb/Mar). Uses QUARTER_MAP lookup.

Restocking Algorithm

Deficit scoring: (forecasted_demand - current_stock) × trend_multiplier × reorder_urgency. Budget allocated top-down by score.

Key Relationships

Inventory ↔ Demand Forecasts

Joined by sku / item_sku. Used in restocking algorithm to compute deficit scores.

Orders ↔ Inventory

Orders reference inventory items in their items[] array. Shared warehouse and category fields.

Backlog ↔ Purchase Orders

Linked by backlog_item_id. Backlog endpoint adds has_purchase_order flag dynamically.

Data Flow

Request Lifecycle

JSON Files → mock_data.py → FastAPI Endpoints → api.js (Axios) → Vue Views → User

Filter Flow

FilterBar UI → useFilters (singleton) → View watch() → getCurrentFilters() → API call + query params → apply_filters() server-side

Restocking Algorithm Flow

inventory.json + demand_forecasts.json → Join by SKU → Deficit Score → Sort by priority → Budget allocation

i18n Flow

LanguageSwitcher → useI18n.setLocale() → t() reactive update → en.js / ja.js → Currency auto-switch (USD/JPY)
